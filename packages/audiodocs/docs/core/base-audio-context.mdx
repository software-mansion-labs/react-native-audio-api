# BaseAudioContext

The `BaseAudioContext` interface acts as a supervisor of audio-processing graphs. It provides key processing parameters such as current time, output destination or sample rate.
Additionally, it is responsible for nodes creation and audio-processing graph's lifecycle management.
However, `BaseAudioContext` itself cannot be directly utilized, instead its functionalities must be accessed through one of its derived interfaces: [`AudioContext`](/core/audio-context), [`OfflineAudioContext`](/core/offline-audio-context).

## Properties

| Name | Type | Description | |
| :----: | :----: | :-------- | :-: |
| `currentTime` | `number` | Returns double value representing an ever-increasing hardware time in seconds, starting from 0. | <ReadOnly /> |
| `destination` | [`AudioDestinationNode`](/core/audio-destination-node) | Returns final output destination associated with the context. | <ReadOnly /> |
| `sampleRate` | `number` | Returns float value representing the sample rate (in samples per seconds) used by all nodes in this context. | <ReadOnly /> |
| `state` | [`ContextState`](/types/context-state) | Returns enumerated value represents the current state of the context. | <ReadOnly /> |

## Methods

### `createAnalyser`

Above method lets you create [`AnalyserNode`](/visualization/analyser-node).

**Returns** `AnalyserNode`.

### `createBiquadFilter`

Above method lets you create [`BiquadFilterNode`](/effects/biquad-filter-node).

**Returns** `BiquadFilterNode`.

### `createBuffer`

Above method lets you create [`AudioBuffer`](/sources/audio-buffer).

| Parameters | Type | Description |
| :---: | :---: | :---- |
| `numOfChannels` | `number` | An integer representing the number of channels of the buffer. |
| `length` | `number` | An integer representing the length of the buffer in sampleFrames. Two seconds buffer has length equals to `2 * sampleRate`. |
| `sampleRate` | `number` | A float representing the sample rate of the buffer. |

Throws `NotSupportedError` if number of channels is outside the range [1, 32], <br />
or sample rate is outside the range [8000, 96000],
or if length is less then 1.

**Returns** `AudioBuffer`.

### `createBufferSource`

Above method lets you create [`AudioBufferSourceNode`](/sources/audio-buffer-source-node).

**Returns** `AudioBufferSourceNode`.

### `createGain`

Above method lets you create [`GainNode`](/effects/gain-node).

**Returns** `GainNode`.

### `createOscillator`

Above method lets you create [`OscillatorNode`](/sources/oscillator-node).

**Returns** `OscillatorNode`.

### `createPeriodicWave`

Above method lets you create [`PeriodicWave`](/effects/periodic-wave).

| Parameters | Type | Description |
| :---: | :---: | :---- |
| `real` | `number[]` | An array of cosine terms. |
| `imag` | `number[]` | An array of sine terms. |
| `constraints` <Optional /> | [`PeriodicWaveConstraints`](/types/periodic-wave-constraints) | An object that specifies if normalization is disabled. |

Throws `InvalidAccessError` real and imag arrays do not have same length.

**Returns** `PeriodicWave`.

### `createStereoPanner`

Above method lets you create [`StereoPannerNode`](/spatialization/stereo-panner-node).

**Returns** `StereoPannerNode`.

### `decodeAudioDataSource`

Above method lets you decode audio data file.

| Parameters | Type | Description |
| :---: | :---: | :---- |
| `sourcePath` | `string` | Path to audio file. |

It is asynchronous method, so it returns a promise.

**Returns** `Promise<AudioBuffer>`.
